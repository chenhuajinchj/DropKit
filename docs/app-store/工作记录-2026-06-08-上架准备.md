## 排查发现（含对自动排查的纠正）

用两个 Explore agent 分别扫了「合规面」和「签名分发面」，但二者都有**过度报警**，已逐条纠正：

| 排查给出的"必须做" | 真实结论 |
|---|---|
| 必加 `NSAccessibilityUsageDescription` 否则被拒 | ❌ macOS 没有这个键，辅助功能权限在系统设置里手动授予；加了无效，不是被拒原因 |
| 必加 `NSPasteboardUsageDescription` 否则被拒 | ❌ macOS 读通用剪贴板不受此键管控，不是被拒原因 |
| 上架前要 notarize 公证 | ❌ 公证只用于 GitHub/DMG 直分发；Mac App Store 上传不需要 |
| 改用 Manual 签名 + fastlane + ExportOptions.plist + CI | ❌ 首次 GUI 提交用不上；Xcode 自动签名最省事 |
| 文件时间戳 reason 用 `C617.1` | ⚠️ 不准；DropKit 读的是用户授权的文件，正确是 `3B52.1` |
| 隐私清单顶层键 `NSPrivacyAccessedAPISets` | ⚠️ 错；正确是 `NSPrivacyAccessedAPITypes` |

**额外抓到一个预存 Bug（重要）**：`project.yml` 用了无效的 `resources:` target 键（XcodeGen 不认），导致资源目录被静默忽略——**连 App 图标（Assets.xcassets）都没被打进包**。pbxproj 里完全没有资源构建阶段是铁证。
