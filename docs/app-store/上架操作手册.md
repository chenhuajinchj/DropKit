# DropKit 上架 Mac App Store 操作手册

> 写给非程序员的逐步指引。本仓库的准备工作（隐私清单、版本号、隐私/支持网页、提交文案）已经做好；
> 这份手册覆盖**账号批下来之后**你要亲手在 Xcode 和 App Store Connect 里点的每一步。
>
> 用到的版本：**DropKit 1.0.6** · Bundle ID：**com.dropkit.DropKit**

---

## 阶段 0 — 现在（账号审核中，等待期）

你已付费，Apple 还在审核开发者身份。这期间**不用干等**，仓库侧准备件已就绪：

- ✅ 隐私清单 `PrivacyInfo.xcprivacy`
- ✅ 版本号统一 1.0.6
- ✅ 隐私政策页 + 技术支持页（已部署到 GitHub Pages，源文件在 `docs/app-store/web/`）
- ✅ 提交文案 `docs/app-store/submission-draft.md`
- ⬜ 截图（需 App 实跑后手动截，见文末"截图清单"，**现在就能截**）
- ✅ 部署两个网页拿到 URL

**怎么知道账号批了？** 你会收到 Apple 邮件；或登录 <https://developer.apple.com/account> 能看到完整会员信息、<https://appstoreconnect.apple.com> 能正常进入。

## 阶段 1 — 账号批了：注册 App ID + 建 App 记录

### 1.1 注册 App ID（Bundle ID）
1. 打开 <https://developer.apple.com/account> → **Certificates, Identifiers & Profiles** → 左侧 **Identifiers** → 点 **+**
2. 选 **App IDs** → **App** → Continue
3. Description 填 `DropKit`；Bundle ID 选 **Explicit**，填 `com.dropkit.DropKit`
4. Capabilities 默认即可（沙盒等在 Xcode 里已配），点 **Continue → Register**

### 1.2 在 App Store Connect 建 App
1. 打开 <https://appstoreconnect.apple.com> → **我的 App（Apps）** → 点 **+** → **新建 App**
2. 填：
   - 平台：勾 **macOS**
   - 名称：`DropKit Clipboard`（`DropKit` 已被占用，后台已用这个展示名创建，并已提交审核）
   - 主要语言：建议 **简体中文** 或 **英语（美国）**（按你主要面向的用户）
   - 套装 ID（Bundle ID）：选刚注册的 `com.dropkit.DropKit`
   - SKU：`dropkit-macos-1`
   - 用户访问权限：完全访问
3. 点创建。

## 阶段 2 — Xcode 签名 + 打包 + 上传

### 2.1 打开项目并设置签名
```bash
cd ~/项目/自己的应用/DropKit/DropKit
xcodegen generate
open DropKit.xcodeproj
```
在 Xcode 里：
1. 左侧选蓝色项目图标 → TARGETS 选 **DropKit** → 顶部标签 **Signing & Capabilities**
2. 勾选 **Automatically manage signing（自动管理签名）**
3. **Team** 下拉里选你的开发者账号（账号批了才会出现）
4. Xcode 会自动生成 "Apple Distribution" 证书和 "Mac App Store" 描述文件——看到没有红色报错就对了

### 2.2 打包（Archive）
1. Xcode 顶部中间的运行目标，选 **Any Mac**（不是 "My Mac"，也没有模拟器）
2. 菜单 **Product → Archive**
3. 等编译完成，自动弹出 **Organizer** 窗口（看到你这次的归档记录）

> 如果 Archive 是灰的：确认目标选的是 Any Mac，且当前 scheme 是 DropKit。

### 2.3 上传到 App Store Connect
1. 在 Organizer 里选中刚打的归档 → 右侧 **Distribute App**
2. 选 **App Store Connect** → **Upload**
3. 签名选项保持默认（自动）→ 一路 Next → **Upload**
4. 上传成功后，等 Apple 处理（几分钟到 1 小时）。处理完，App Store Connect 对应版本的"构建版本"里会出现这个 build。

> 出口合规加密那道问题：已在 Info.plist 写了 `ITSAppUsesNonExemptEncryption=false`，正常不会再弹。

## 阶段 3 — 填资料 + 提交审核

进入 App Store Connect → 你的 App → 左侧选版本（1.0.6）：

### 3.1 版本信息（文案直接抄 `submission-draft.md`）
- **App 名称 / 副标题 / 描述 / 关键词**：用 submission-draft 里的内容
- **What's New（本版本新增）**：用 submission-draft 的 1.0.6 段落
- **App 预览和截图**：上传截图（见文末清单）
- **技术支持 URL**：填 `https://chenyuxiaojin.github.io/DropKit/support.html`
- **隐私政策 URL**：填 `https://chenyuxiaojin.github.io/DropKit/privacy.html`
- **构建版本**：选阶段 2 上传的那个 build

### 3.2 App 隐私（App Privacy）
1. 左侧 **App 隐私** → **开始 / 编辑**
2. "你或你的第三方合作伙伴是否从这个 App 收集数据？" → 选 **否（No）**
   （DropKit 不联网、不收集任何数据，理由见隐私政策页）

### 3.3 其他必填
- **年龄分级**：做问卷，DropKit 全选无 → 4+
- **价格与销售范围**：选免费或定价；选发售地区
- **App 审核信息**：
  - 联系人姓名 / 电话 / 邮箱（填你自己的）
  - **备注（Notes）**：把 submission-draft 里的 **Review Notes** 整段贴进去——重点说明"辅助功能权限仅用于可选的摇动手势，核心功能无权限也能用"。这条能减少因辅助功能权限被问询/打回的概率。

### 3.4 提交
右上角 **添加以供审核 / Submit for Review**。之后状态变 "等待审核 → 正在审核"。macOS App 审核通常 1～3 天。

## 常见被拒原因与应对（DropKit 相关）

| 风险点 | 为什么 | 应对 |
|--------|--------|------|
| 辅助功能权限 | 审核员可能问为什么要全局监听 | Review Notes 已说明仅用于可选摇动手势 + 核心功能无权限可用；代码会在无权限时拒绝启动全局拖拽/摇晃监听 |
| 剪贴板监控 | 可能被问数据去向 | 隐私政策页已写明纯本地、不上传；代码默认跳过常见密码管理器和带隐藏标记的剪贴板内容 |
| 沙盒文件访问 | 监控文件夹的权限 | 已用 user-selected + 安全书签，仅访问用户选的文件夹 |
| 截图与实际不符 | 截图必须是真实界面 | 用真机真实界面截图，别用拼图/示意图 |
